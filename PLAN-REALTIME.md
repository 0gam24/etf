# PLAN — 한투 OpenAPI 실시간 시세 활용 기획

> 작성: 2026-05-11
> 배경: 메인페이지 "오늘 뜨는 ETF" 카피 + 5/10(일) 발행일 + KRX 5/8 마감 데이터의 부조화. data.go.kr 무료 API는 일별 마감 데이터만 제공 (장중 tick 데이터 X). 한투 OpenAPI는 무료 + 장중 실시간 시세 + 시세 조회 권한만으로 충분.

---

## 1. 핵심 원칙

| 원칙 | 의미 |
|---|---|
| **시세는 실시간 / 분석은 일별** | 시세(현재가·등락·거래량)는 한투 API 분단위. 분석 글(pulse·surge·flow·income·breaking)은 그대로 일별 마감 데이터 기반 — 정합성 유지 |
| **장 상태 정직 표기** | 장중 = "실시간 · HH:MM:SS 갱신" / 장 마감 = "오늘 종가 · 15:30 마감" / 휴장 = "마지막 거래일 N월 N일 기준" |
| **폴백 안전** | 한투 API 실패 → data.go.kr 일별 데이터 자동 폴백. UX 단절 없음 |
| **호출 절약 = 캐시 우선** | 한투 분당 20회 제한. edge 캐시 (장중 30초·장 마감 30분·휴장 24h)로 호출 ≤ 분당 5회 유지 |
| **시세만 사용** | 매매 권한 신청 X. 분석 글 본문 안의 권유 표현 차단 정책(YmylGuard)도 그대로 |

## 2. 활용처 (페이지별 우선순위)

### Phase 1 — 메인페이지 (가장 큰 visual impact · 즉시 적용)
| 컴포넌트 | 현재 | 한투 API 적용 후 |
|---|---|---|
| `HomeHeroV3` "오늘 거래량 1위" 카드 | 5/8 종가 14,365원 ▼6.72% | **장중**: 14,500원 ▲0.94% (14:32:15 갱신) / **마감 후**: 5/11 종가 / **휴장**: 5/8 기준 명시 |
| `EtfMarketPulse` 거래량 TOP10 / 상승·하락 TOP5 | 30분 캐시 (마감 데이터) | 장중 30초 갱신, 마감 후 동결 |
| `HomeSnapshot` 시장 평균 등락 | 마감 데이터 평균 | 실시간 평균 |
| 헤드라인 카피 | "오늘 뜨는 ETF, 왜 오르는지까지" | 그대로 + 부제 "**장중 실시간 · KRX 5/11 14:32 기준**" 동적 |
| 발행일 표시 ("5월 10일 (일) · 1시간 전 발행") | 글 publishedAt | 글 publishedAt 그대로 (다른 칸) + 시세 갱신 시각 별도 |

### Phase 2 — 종목 사전 `/etf/{slug}` (1099 페이지)
- Hero 영역의 현재가·등락·거래량·시가·고가·저가 stats — 한투 API 호출
- 페이지 접속 시점에 fetch (lazy). 1099개 모두 prefetch 안 함 (호출 폭증 회피).
- 캐시: 종목별 30초 (장중) / 30분 (마감)

### Phase 3 — 글 페이지 ticker 칩
- 글 본문 byline 옆 ticker 칩 (`/etf/{ticker}` link) 옆에 현재가 mini display
- 예: `[KODEX 방산TOP10 · 14,500원 ▲0.94%]`
- 캐시: 30초

### Phase 4 — HoldingsPanel 구성종목 시세 (선택)
- ETF 구성종목 TOP 10의 개별 종목 시세 (한화에어로스페이스·현대로템 등 개별 주식)
- 한투 OpenAPI는 ETF뿐 아니라 개별 주식도 지원
- 호출 폭증 위험 → 사용자 클릭으로 펼침 시에만 fetch

## 3. 데이터 흐름

```
┌─────────────────────────────────────────────────────────────┐
│ 빌드 시점 (cron 평일 16:00 KST)                              │
│  data.go.kr 일별 → DataMiner → 분석 글 7편 → MDX → git push │
│  → Cloudflare 자동 빌드 → 정적 글 페이지                     │
└─────────────────────────────────────────────────────────────┘
                          │
                          ↓ 시청자가 페이지 열면
┌─────────────────────────────────────────────────────────────┐
│ 런타임 (시청자 페이지 로드)                                   │
│  page.tsx SSR / 클라이언트 컴포넌트                          │
│     ↓                                                        │
│  /api/etf/realtime route handler                            │
│     ├─ access_token 캐시 (Cloudflare KV, 24h)               │
│     ├─ 한투 시세 호출 (분당 5회 제한)                        │
│     ├─ edge 캐시 (장중 30s / 마감 30min / 휴장 24h)         │
│     └─ 실패 시 data.go.kr 폴백                              │
│     ↓                                                        │
│  시청자 화면: "장중 14:32:15 갱신 / 14,500원 ▲0.94%"        │
└─────────────────────────────────────────────────────────────┘
```

## 4. API 호출 예산

- 한투 분당 한도: **20호출** (시세 endpoint 기준)
- 메인페이지 1회 로드 = TOP10 종목 시세 = 10호출
- 캐시 30초로 동일 종목 호출 1회/30초 → 분당 2호출
- 종목 사전 페이지 1회 로드 = 1호출
- 평균 부하 (예상):
  - 메인페이지 동시 시청자 100명 + edge 캐시 30초 → 분당 2회 (캐시 hit)
  - 종목 사전 동시 50명 + 각 다른 종목 → 분당 50회 (위험)
  - **mitigation**: 종목 사전은 캐시 시간 60초로 늘리고, 동일 종목 분당 1회 호출 보장

## 6. 구현 단계 (작업 순서)

### Step 1 — 인프라
- `src/lib/kis.ts` — 한투 API 클라이언트 (access_token 자동 발급/갱신, 호출 메서드, rate limit)
- `src/app/api/etf/realtime/route.ts` — Next.js Route Handler (edge 캐시 + 폴백)
- 환경변수 docs `.env.example` 갱신

### Step 2 — 메인페이지 (Phase 1)
- `HomeHeroV3` 컴포넌트: topEtf 시세 부분만 클라이언트 컴포넌트로 분리 + `useSWR` (30초 polling)
- `EtfMarketPulse` 컴포넌트: 기존 `/api/etf` → `/api/etf/realtime` 으로 전환 + 갱신 시각 표기
- 시장 상태 분기 카피 (`장중` / `마감 후` / `휴장`)

### Step 3 — 종목 사전 (Phase 2)
- `/etf/{slug}` Hero stats를 클라이언트 컴포넌트로 분리
- 페이지 진입 시 1회 fetch + 30초마다 silent refresh (장중에만)

### Step 4 — 글 페이지 ticker 칩 (Phase 3)
- ticker 칩 옆에 mini price display (선택, 깔끔하지 않으면 보류)

### Step 5 — 폴백·모니터링
- 한투 API 응답 실패 logging
- daily-pulse cron Summary에 한투 호출 통계 추가

## 7. 위험 + 완화

| 위험 | 완화 |
|---|---|
| 한투 API 키 만료/회수 | `data.go.kr` 자동 폴백 (사용자 알림 없이 매끄럽게) |
| 분당 호출 한도 초과 | edge 캐시 + 종목별 호출 throttle |
| `access_token` 24h 만료 | Cloudflare KV 또는 메모리에 토큰 + 만료 30분 전 자동 재발급 |
| 한투 서버 장애 | data.go.kr 폴백 + 시청자에게 "마감 데이터 표시 중" 작은 라벨 |
| 장 시간 외 API 호출 낭비 | 시장 상태 체크 후 마감 후·휴장에는 캐시 길게 |
| 사용자 키 유출 | env 변수 only, 코드/로그/commit 노출 X |

## 8. 메인페이지 카피 정직화 (한투 API 도입 전이라도 즉시 적용 가능)

현재:
```
DAILY ETF PULSE
5월 10일 (일) · 1시간 전 발행
오늘 뜨는 ETF, 왜 오르는지까지
맥락과 근거로 설명합니다.
```

개선 (한투 API 도입 전):
```
DAILY ETF PULSE
2026-05-08 종가 기준 · 5월 10일(일) 발행
이번 거래일 거래량 TOP, 왜 움직였나
맥락과 근거로 설명합니다.
```

개선 (한투 API 도입 후 · 장 상태별 분기):
- **장중 (KST 09:00~15:30)**: `"장중 실시간 · 14:32:15 갱신"` + "오늘 뜨는 ETF, 왜 오르는지까지"
- **장 마감 후**: `"오늘 종가 기준 · 5/11 15:30 마감"` + "오늘 뜨는 ETF, 왜 움직였나"
- **휴장**: `"마지막 거래일 5/8(금) 기준"` + "이번 거래일 거래량 TOP"

## 9. 결정 필요 사항

1. **Phase 1만 먼저 (메인페이지 시세 위젯) → Phase 2~4는 검증 후 단계 적용** vs **전체 Phase 한꺼번에**
2. **시세 갱신 주기**: 장중 30초 (권장) vs 1분 vs 5초 (대시보드급)
3. **모의투자 키 / 실전 키**: 시세 조회만이면 둘 다 동일, **실전 키 권장** (모의는 가끔 모의 시세 반환)
4. **카피 정직화는 한투 API 도입 전 즉시 적용?** (작업 1~2시간, 메인페이지·종목사전 부조화 해소)

## 10. 즉시 시작 가능한 작업 (한투 키 발급 대기 중에도)

A. **메인페이지 카피 정직화** (한투 API 무관 · 1~2시간)
- 부제에 baseDate 명시
- 시장 상태 분기 (장중·마감 후·휴장)
- 발행일 표기를 baseDate 기준으로 변경

B. **`src/lib/kis.ts` 스켈레톤** (한투 키 없이 작성 가능)
- 인터페이스 정의
- 폴백 로직
- 캐시 전략 구현

C. **`/api/etf/realtime` route 스켈레톤**
- mock 응답으로 동작 확인
- 한투 키 등록 후 실제 호출로 전환

키 발급 대기하는 동안 A·B·C 진행하면 키 등록 직후 즉시 활성화 가능.
